ECMAScript 标准

Created2019-01-02|Updated2025-02-23|Development
|Word Count:1.6k

ECMAScript 全称 European Computer Manufacturers Association Script，简称 ES，是一种由 Ecma 国际组织标准化的脚本语言规范，它是 JavaScript 的标准化版本，定义了语言的核心特性

先决条件

阅读本文之前需要熟练掌握 HTML / CSS / JS 网页基础知识 点击跳转

变量的声明

在 JavaScript 中，var / let / const 都用于声明变量，但它们的行为和作用域有所不同

使用 var 声明

具有函数作用域，如果在函数内声明，变量仅在函数内有效；如果在函数外声明，则为全局变量

变量声明会被提升到函数或全局作用域的顶部，但赋值不会提升

允许在同一作用域内重复声明

使用 let 声明

具有块级作用域，变量仅在声明它的块 {} 内有效

let 声明同样会被提升，但在初始化之前会出现暂时性死区，访问会抛出 ReferenceError

let 不允许在同一作用域内重复声明

使用 const 声明

let 的常量形式，使用 const 在声明时必须初始化，且不能重新赋值，但对于对象和数组，可以修改其属性和元素

暂时性死区

暂时性死区（Temporal Dead Zone, TDZ）是指从进入作用域到变量声明之间的区域，在这段时间内访问变量会抛出 ReferenceError。let 和 const 声明的变量在声明之前是不可访问的，即使它们已经被提升

TDZ 的开始：进入块级作用域 {} 时，num 的声明被提升，但此时变量处于未初始化状态

TDZ 的结束：执行到 let num = 1 时，变量被初始化，TDZ 结束，之后可以正常访问

为什么会有暂时性死区？

暂时性死区的设计是为了避免变量在声明之前被意外使用，从而减少潜在的错误。这种行为使得 JavaScript 更加严谨，尤其是在大型项目中，可以帮助开发者更容易地发现和修复问题

Promise

Promise 是 JavaScript 中用于处理异步操作的对象。它代表一个异步操作的最终完成（或失败）及其结果值。Promise 提供了一种更优雅的方式来处理异步代码，避免了传统的回调地狱（Callback Hell）

ES6+ 简化语法

ES6 之后引入了许多简化语法，提升了代码的可读性和简洁性

箭头函数

箭头函数（Arrow Function）是 JavaScript 中一种简洁的函数定义方式，使用 => 语法来定义函数

当参数只有一个时，可以省略 ()

特点

不会创建自己的 this，而是继承外层作用域的 this

不能使用 call / apply / bind 改变 this 指向

不能使用 new 关键字调用，因为它没有 Construct 内部方法

没有自己的 arguments 对象

可以通过剩余参数（Rest Parameters）获取参数列表

模板字符串

模板字符串（Template Literals）是一种允许嵌入表达式的字符串字面量。它们使用反引号而不是单引号或双引号来定义。模板字符串提供了更强大的字符串操作功能，包括多行字符串和字符串插值

解构

解构赋值（Destructuring Assignment）允许你从数组或对象中提取值，并将其赋给变量。解构赋值使得代码更简洁、易读

数组解构

数组解构允许你从数组中提取值，并按顺序赋值给变量
