Hexo-Cli 操作指南

部署到 GitHub Page

本示例假设您在本地环境已经成功创建项目，并且拥有自己的 GitHub 账号，着重讲述如何将本地的项目部署到 GitHub Page

第一步、新建一个仓库

进入 GitHub 创建仓库的页面

仓库名称需要填写 [username].github.io，这是 GitHub Page 的固定格式，将 [username] 替换成自己账号的用户名

填写完成后，点击下面的 Create repository 按钮完成创建

第二步、设置个人访问令牌

个人访问令牌 (Personal Access Token, PAT) 是 GitHub 提供的一种身份验证方式，主要用于增强账户的安全性，并允许用户通过 API 或命令行工具与 GitHub 进行交互

进入用户设置页面

点击左侧菜单底部的 Developer settings 按钮

点击 Tokens (classic) 按钮，再点击 Generate new token (classic) 按钮，进入 Token 的编辑页面

Note 输入框填写这个 Token 的用途；Expiration 选择到期时间，图中的示例是永不到期

Select scopes 根据自己实际需求，选择允许在哪些作用域下访问个人令牌，图中的示例是全选

最后，点击下面的 Generate token 按钮来生成 Token

需要将最终生成的 Token 保存到本地，用于项目推送

第三步、创建 SSH Key

GitHub SSH Key 是用于通过 SSH 协议与 GitHub 进行安全通信的一种身份验证方式。SSH (Secure Shell) 是一种加密的网络协议，用于在不安全的网络中安全地进行远程登录和文件传输

终端输入 ssh-keygen -t rsa 命令后 Enter 来创建 SSH Key

运行成功后，会在 C:\Users\[username]\.ssh 生成 id_rsa.pub

进入 GitHub 用户设置页面

点击左侧菜单 SSH and GPG keys 按钮，再点击 New SSH key 按钮

进入 SSH Key 编辑页面，将 id_rsa.pub 内容复制到 Key 文本域

最后点击 Add SSH key 按钮完成创建

第四步、推送到远端

在项目下执行 hexo d

首次推送，会出现登录提示

输入您的 GitHub 账号和密码，点击 Login 按钮

在终端输入用户名和 PAT，Enter

再次强调，Password for ‘https://lphcev@github.com’: 这段不是输入 GitHub 密码，而是个人访问令牌的 Value

当终端出现如下信息，代表已经推送成功

经过了上述一系列配置，之后的每次推送只需在项目下执行 hexo d 即可完成，无需额外输入用户认证信息
