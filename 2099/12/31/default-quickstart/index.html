X-Blog 快速入门

Created2099-12-31|Updated2024-11-23|Default

[置顶] X-Blog 是一个基于 Hexo + Butterfly 搭建的个人博客站点；Hexo 是一个快速、简单且强大的博客框架，它使用 Markdown 来编写内容，并生成静态网页。本篇文章主要讲述 X-Blog 的基本使用

温馨提示：如果您当前的屏幕分辨率超出了 1920 x 1080，建议将浏览器窗口缩放为 125% 视觉效果会更佳

快捷键：Ctrl + 鼠标滚轮

环境搭建

安装

本教程不再展开去讲从零开始搭建博客，因为在官网已经有很详细的说明，参见如下

  • Hexo：https://hexo.io/
  • Butterfly：https://butterfly.js.org/

以下列出部分安装命令

如果安装的是最新版本，可能会出现不稳定或版本冲突的情况（例如使用 npm 安装某个依赖包报 err）。以下展示本博客用的所有相关依赖版本，更新于 2024.11.20 之前，如有需要，可以通过 npm install [package]@[version] 的形式来指定安装的版本号 （Node.js 中文网：https://nodejs.p2hp.com/）

主依赖

名称 版本 描述
npm 10.9.0 包管理工具
hexo-cli 4.3.2 hexo 命令行执行
Hexo 7.3.0 博客基础框架
Butterfly 5.2.0 主题风格

其他依赖项可以参考 package.json

部署到 GitHub Page

GitHub Pages 是 GitHub 提供的一项静态站点托管服务，它允许用户直接从 GitHub 上的仓库获取 HTML、CSS 和 JavaScript 文件，然后发布网站。将您的项目部署到 GitHub Page，让您随时随地都能阅览文章

第一步、新建一个仓库

进入 GitHub 创建仓库的页面

仓库名称需要填写 [username].github.io，这是 GitHub Page 的固定格式，将 [username] 替换成自己账号的用户名

填写完成后，点击下面的 Create repository 按钮完成创建

第二步、设置个人访问令牌

个人访问令牌 (Personal Access Token, PAT) 是 GitHub 提供的一种身份验证方式，主要用于增强账户的安全性，并允许用户通过 API 或命令行工具与 GitHub 进行交互

进入用户设置页面

点击左侧菜单底部的 Developer settings 按钮

点击 Tokens (classic) 按钮，再点击 Generate new token (classic) 按钮，进入 Token 的编辑页面

Note 输入框填写这个 Token 的用途；Expiration 选择到期时间，图中的示例是永不到期

Select scopes 根据自己实际需求，选择允许在哪些作用域下访问个人令牌，图中的示例是全选

最后，点击下面的 Generate token 按钮来生成 Token

需要将最终生成的 Token 保存到本地，用于项目推送

第三步、创建 SSH Key

GitHub SSH Key 是用于通过 SSH 协议与 GitHub 进行安全通信的一种身份验证方式。SSH (Secure Shell) 是一种加密的网络协议，用于在不安全的网络中安全地进行远程登录和文件传输

终端输入 ssh-keygen -t rsa 命令后 Enter 来创建 SSH Key

运行成功后，会在 C:\Users\[username]\.ssh 生成 id_rsa.pub

进入 GitHub 用户设置页面

点击左侧菜单 SSH and GPG keys 按钮，再点击 New SSH key 按钮

进入 SSH Key 编辑页面，将 id_rsa.pub 内容复制到 Key 文本域

最后点击 Add SSH key 按钮完成创建

第四步、推送到远端

在项目下执行 hexo d

首次推送，会出现登录提示

输入您的 GitHub 账号和密码，点击 Login 按钮

在终端输入用户名和 PAT，Enter

再次强调，Password for ‘https://lphcev@github.com’: 这段不是输入 GitHub 密码，而是个人访问令牌的 Value

当终端出现如下信息，代表已经推送成功

经过了上述一系列配置，之后的每次推送只需在项目下执行 hexo d 即可完成，无需额外输入用户认证信息

Hexo 常用命令

hexo new [layout] <title>

创建命令，layout 指定布局，title 指定主标题。可以简写成 hexo n

hexo clean

用于清理缓存文件，即删除 public 文件夹。

hexo generate

用于生成静态文件，执行后在 Hexo 站点根目录下生成 public 文件夹。可以简写成 hexo g

hexo deploy

用于部署项目。可以简写成 hexo d

hexo server

用于启动本地服务器，默认端口为 4000。可以简写成 hexo s

关于 layout

在创建时，layout 是可选参数，用于指定布局，常用布局有以下两种

  • post 创建普通的博客文章。它通常包含标题、日期、分类、标签、摘要和正文等部分
  • page 创建页面，如关于页面、联系页面等。页面与文章的主要区别在于它们在网站结构中的位置和用途。页面通常不包含在博客文章的列表中，而是作为网站的独立部分存在

Markdown

Markdown 是一种轻量级标记语言，允许使用简单的文本格式来编写具备特定排版格式的文档。Markdown 的核心理念是“重内容轻形式”，即让写作者能够专注于内容本身，而无需过多纠结于排版细节。它通过简单的标记语法，使普通文本内容具有一定的格式，从而提升了工作效率

语法规则

Markdown 的语法规则相对简单且直观，主要包括以下元素

标题

通过在单词或短语前面添加井号 # 来创建，# 的数量代表了标题的级别。通常，一级标题用于整篇文章的标题（但很多文章选择省略一级标题，直接使用二级标题作为根标题），后续标题等级依次递减，用于文章内部的小节划分

列表

无序列表通过在每个列表项前面添加破折号 -、星号 * 或加号 + 来创建

有序列表则在每个列表项前添加数字并紧跟一个句点

超链接和图片

通过特定的语法来插入网页链接和图片

“图片描述”相当于注释，不会在网页中显示，“链接描述”是用户点击后跳转到的目标地址的文本描述

引用

使用符号 > 来实现，可以单独划出一块编辑空间，与正文形成明显的划分。引用块内部可以嵌套其他 Markdown 元素，如标题、列表、代码等。在引用块内部再次使用 > 符号进行嵌套引用，每嵌套一层，> 符号数量增加

代码

使用反引号 (`) 将代码内容包裹起来，或使用三个反引号 (```) 包围代码块，并可指定编程语言以获得合适的高亮

文本强调

使用星号 * 或下划线 _ 将文本设置为粗体或斜体来强调其重要性

分隔符

单独一行上使用三个或多个星号 ***、破折号 --- 或下划线 ___ 来创建分隔线

表格

使用三个或多个连字符 --- 分割标题行和内容行，并使用管道 | 分隔每列

在 Hexo 中，标签插件（或称为外挂）是用于增强站点功能的重要组件。这些插件可以添加新的功能、改进现有的功能或与第三方服务集成，接下来介绍几种项目中常用的标签外挂
