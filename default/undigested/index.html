闭包

JavaScript 中闭包（Closure）是指一个函数能够访问并记住其词法作用域，即使该函数在其词法作用域之外执行。简单来说，闭包允许函数访问其定义时所处的环境中的变量，即使这个函数在定义环境之外被调用

闭包形成条件

  1. 函数嵌套

    • 一个函数（内部函数）定义在另一个函数（外部函数）的内部
    • 内部函数可以访问外部函数的变量和参数

  2. 内部函数被外部函数以外的作用域引用

    • 内部函数必须被外部函数以外的作用域引用（例如，被返回、赋值给全局变量、作为回调函数传递等）
    • 只有这样，内部函数才会保留对外部函数作用域的引用，从而形成闭包

  3. 闭包的关键点

    • 词法作用域：闭包是基于词法作用域的，即函数在定义时就已经确定了它可以访问的作用域
    • 作用域链：内部函数会保留对外部函数作用域的引用，即使外部函数已经执行完毕
    • 内存管理：闭包会阻止外部函数的作用域被垃圾回收，因为内部函数仍然持有对它的引用

call \ apply \ bind

在 JavaScript 中，call \ apply \ bind 是用于改变函数执行时 this 指向的方法。它们都可以显式地设置函数内部的 this 值。call 方法允许你调用一个函数，并显式地指定函数内部的 this 值

call 方法还可以传递参数列表

apply 方法与 call 类似，但它接受一个参数数组，而不是单独的参数列表

bind 方法会创建一个新的函数，并将 this 值绑定到指定的对象。与 call 和 apply 不同，bind 不会立即执行函数，而是返回一个新的函数，可以在稍后调用

总结

  • 参数数量固定时，使用 call

  • 参数数量不固定时，使用 apply

  • 需要创建一个新的函数并延迟执行时，使用 bind

防抖（Debebounce）和节流（Throttle）是用于控制高频事件触发频率的优化手段

柯里化

柯里化函数（Currying Function）的作用是将一个多参数函数转换为一系列单参数函数，使得函数可以分步接收参数，直到所有参数都传递完毕后再执行
